# 人物表演方案生成器

## 阶段定位

第四阶段负责从案例库中召回相关人物表演，回答“哪些既有案例值得参考”。第五阶段负责把剧情判断、角色资产、项目约束与召回案例合成为一套可以直接用于图片或视频生成的表演方案，回答“这个镜头具体应该怎样演”。

第一版使用本地可解释引擎，不调用真实大模型。`requestPerformancePlansFromModel()` 已作为云端 AI 接口保留，后续可以在不改变页面交互的前提下替换本地生成函数。

## 输入信息

生成器综合使用：

- 剧情或画面描述。
- 当前角色的气质、情绪表现和禁止项。
- 当前项目风格母版与默认负面约束。
- 剧情刺激、真实情绪、外在面具、人物基线和权力位置。
- 镜头景别和镜头时长。
- 用户主动选择的案例和自动召回的高分案例。

## 三种策略

### 克制留白

适合压制、冻结和隐忍阶段。人物尽量维持表面稳定，只允许眼神、呼吸、下颌或手部出现少量泄露。

### 递进泄露

按照“刺激进入、控制松动、做出决定、重新收束”的顺序建立清晰变化，适合需要观众看懂人物心理转折的镜头。

### 临界释放

让情绪接近爆发边缘，允许局部肌肉张力明显增强，但限制五官失控和多余肢体动作，适合冲突峰值前一拍。

引擎会根据当前表演阶段、参考案例数量和案例质量计算适配度，并自动把更匹配的方案排在前面。

## 分秒时间轴

时间轴会根据镜头时长自动调整：

- `0.5-2s`：两拍，刺激进入与反应泄露。
- `2-4s`：三拍，刺激进入、控制与泄露、决定与收束。
- `4-10s`：四拍，额外保留余韵阶段。

每一拍都包含明确时间范围、表演阶段和可观察动作，避免在短镜头中堆叠过多连续反应。

## 表情与身体通道

引擎会从召回案例中提取：

- 眼神与眼睑。
- 眉部。
- 唇部与口周。
- 下颌与咬肌。
- 呼吸与鼻翼。
- 颈部与吞咽。
- 手部。
- 肩颈与身体姿态。
- 发声准备。

训练数据中的中文通道名称会映射到统一结构，再从案例动作中提取对应短句。找不到可用案例时才使用保守的专业兜底描述。

## 工作台操作

```text
输入剧情和场景
  -> 设置角色、景别和镜头时长
  -> 查看并选择人物微表情参考
  -> 点击“生成表演方案”
  -> 比较三套导演策略
  -> 采用并编辑其中一套
  -> 需要保留时锁定
  -> 复制或保存词条
```

采用方案后，完整内容会进入【微表情】模块卡片。编辑器中的修改会立即同步到该卡片。输入发生变化时，页面会提示重新生成；锁定方案在重新生成时保持原内容。

## 保存与再次召回

点击“保存词条”会创建一个状态为 `published` 的 `expression_case`：

- 保存完整导演方案作为中文内容。
- 保存精简的可见动作，供以后召回使用。
- 保存刺激、真实情绪、面具、阶段、时长、景别和通道结构。
- 保存来源案例 ID 和镜头位置，便于追溯。

新词条会直接加入用户案例池，后续输入相似剧情时可以再次被召回。

## 数据兼容

新增状态位于 `workbench.performancePlan`，通过 `mergeState()` 增量补齐。旧版 `localStorage` 中的角色、项目、存储词条、提示词和结果卡片不会被覆盖。
